SIS-FP Project

Control y Calidad del Software

Introducción

El SIS-FP Project es una solución integral para la gestión de seguridad portuaria desarrollada bajo criterios de calidad y buenas prácticas del software. Esta landing presenta los instrumentos de documentación y control de calidad aplicados al proceso de desarrollo del sistema, siguiendo los lineamientos de la evidencia GA11-220501098-AA1-EV03 y los fundamentos de calidad establecidos en normas como ISO 25000 y CMMI.

Buenas Prácticas de Calidad Aplicadas

Buena Práctica Marco de Referencia Descripción Beneficio
Revisión por pares de código CMMI / ISO 9001 Validación colaborativa del código antes de su integración final. Reduce errores tempranos y mejora la mantenibilidad.
Control de versiones SCRUM / DevOps Registro y trazabilidad de cambios en el código fuente. Facilita el trabajo colaborativo y evita conflictos.
Pruebas automatizadas ISO 25000 Verificación automática de la funcionalidad del sistema SIS-FP. Disminuye tiempo de prueba y aumenta confiabilidad.
Documentación estandarizada CMMI Uso de formatos únicos para registrar procesos y resultados. Mejora la trazabilidad y facilita auditorías.
Retroalimentación continua SCRUM Evaluación constante de avances del software. Mejora la comunicación y asegura la calidad final.

Conclusiones

La aplicación de buenas prácticas de calidad dentro del SIS-FP Project permitió estandarizar el proceso de desarrollo, documentar cada fase del proyecto y fortalecer la confiabilidad del sistema de seguridad portuaria. Los instrumentos de calidad diseñados facilitan la trazabilidad, la comunicación entre equipos y la mejora continua, asegurando que el producto final cumpla con los requisitos del cliente y las normas internacionales de calidad.
